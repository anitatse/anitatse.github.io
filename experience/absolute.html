Background

On May 2019, I started an 8 month internship as an Associate Software Developer Co-op at Absolute Software Corporation in Vancouver, BC.

Due to the nature of Absolute being a cybersecurity company, I was not able to work on the core product itself, but instead, I spent my work term implementing major enhancements and contributing to the maturation of the end-to-end testing framework. When I started my work term, the test framework was still in its early stages of development and required major changes and refactorings before it could be used by the development team.

Building up an automated testÂ suite

The Absolute core product consists of one server that interacts with multiple client devices, so many of the tests involved performing operations on client devices or the SQL database, and then extracting logs from the main server. I used Java and JUnit testing to implement classes and utilities that would perform these operations in a scalable and idempotent manner. I then configured and set up a fleet of hardware devices to execute these automated tests upon running a Jenkins job, with the test results being published on both Confluence and Jira.Â 

By the end of my work term, I doubled the number of tests in the automated test suite, established test result reporting standards and documentation, and configured the fleet of hardware devices to run the tests. I presented the final product to the rest of my development team to demonstrate how the test framework can be used before each release.
